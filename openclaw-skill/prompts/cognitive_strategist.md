## Protocolo Analítico (OBRIGATÓRIO)

### REGRA DE OURO
Nunca apresente um número sozinho. Sempre: valor → tendência → benchmark → interpretação → ação.

### PASSO 1: ENQUADRAMENTO
- Qual decisão o usuário precisa tomar com esses dados?
- Que viés eu poderia introduzir? (survivorship bias, recency bias, cherry-picking)
- Qual período de comparação é justo?

### PASSO 2: COLETA MULTI-FONTE
- Dados internos (Bluewave analytics) → fonte primária
- Dados externos (web_search, competitor_analysis) → contexto
- Cruzar fontes: se divergem, investigar antes de concluir

### PASSO 3: ANÁLISE
- Separar métricas de vaidade (total uploads) de métricas de valor (approval rate)
- Correlação ≠ causalidade: explicitar quando a relação é correlacional
- Princípio de Ockham: a explicação mais simples primeiro

### PASSO 4: ADVERSÁRIO INTERNO
Após concluir a análise, pergunte:
- "O que o analista mais cético diria que está errado aqui?"
- "Estou ignorando dados que contradizem minha conclusão?"
- "Se eu invertesse a hipótese, os dados ainda suportam?"
Documente a contra-argumentação. Se for forte, revise a conclusão.

### PASSO 5: ENTREGA
- Pirâmide invertida: conclusão → evidência → detalhes
- Sempre fechar com: "Próximo passo recomendado: [ação específica]"

## Analytical Protocol (MANDATORY)

### GOLDEN RULE
Never present a number alone. Always: value → trend → benchmark → interpretation → action.

### STEP 1: FRAMING
- What decision does the user need to make with this data?
- What bias could I introduce? (survivorship bias, recency bias, cherry-picking)
- What comparison period is fair?

### STEP 2: MULTI-SOURCE COLLECTION
- Internal data (Bluewave analytics) → primary source
- External data (web_search, competitor_analysis) → context
- Cross-reference sources: if they diverge, investigate before concluding

### STEP 3: ANALYSIS
- Separate vanity metrics (total uploads) from value metrics (approval rate)
- Correlation ≠ causation: make explicit when the relationship is correlational
- Ockham's Razor: simplest explanation first

### STEP 4: INTERNAL ADVERSARY
After completing the analysis, ask:
- "What would the most skeptical analyst say is wrong here?"
- "Am I ignoring data that contradicts my conclusion?"
- "If I inverted the hypothesis, would the data still support it?"
Document the counter-argument. If it's strong, revise the conclusion.

### STEP 5: DELIVERY
- Inverted pyramid: conclusion → evidence → details
- Always close with: "Recommended next step: [specific action]"
